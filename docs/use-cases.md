# Use Cases

ScanReady 1.0 is designed for workflows where high-poly scans need to become lighter, cleaner, and easier to use.

Raw 3D scans are often too heavy for realtime production. They may look detailed, but they can be difficult to move, export, render, or use inside VR and game engines.

ScanReady 1.0 helps convert those scans into optimized baked assets while preserving the visual identity of the original capture.

---

## VR and AR

VR and AR experiences require stable performance.

A very dense scan can be too heavy for smooth interaction, especially on standalone headsets or realtime viewers.

ScanReady 1.0 helps prepare scanned models for VR and AR by:

- Reducing polygon count.
- Creating a lighter mesh.
- Baking visual detail into textures.
- Supporting Normal and Ambient Occlusion maps.
- Helping keep the asset easier to render.
- Making the model more practical for interactive scenes.

This is useful for virtual museums, training simulations, cultural heritage experiences, product visualization, and immersive environments.

---

## Videogames

Game engines need assets that balance visual quality and performance.

A raw scan with millions of polygons is rarely suitable for direct use in a videogame.

ScanReady 1.0 helps prepare game-ready assets by:

- Creating an optimized lowpoly version.
- Generating UVs.
- Preparing a bake cage.
- Baking texture information from the high-poly scan.
- Saving texture maps for external workflows.
- Keeping detail in textures instead of heavy geometry.

This makes scanned props, environment pieces, and real-world objects easier to bring into a game pipeline.

---

## Realtime Visualization

Realtime visualization requires assets that can be inspected, moved, and rendered smoothly.

ScanReady 1.0 is useful when you need to show scanned objects in:

- Interactive product viewers.
- Architecture previews.
- Web-based 3D viewers.
- Blender realtime scenes.
- Realtime rendering tools.
- Lightweight review scenes.

The addon helps reduce scene weight while keeping the object visually recognizable.

---

## Cultural Heritage

Cultural heritage scans often contain valuable surface detail, but the raw data can be very heavy.

ScanReady 1.0 can help create lighter versions for:

- Virtual exhibitions.
- Museum previews.
- Educational experiences.
- Online collections.
- Interactive archives.
- VR historical reconstructions.

The goal is not to replace the original scan archive.

The goal is to create a practical version that can be used for presentation, interaction, and realtime exploration.

---

## Photogrammetry Assets

Photogrammetry often produces detailed but dense meshes.

ScanReady 1.0 helps process photogrammetry models by:

- Simplifying the captured geometry.
- Creating usable UVs.
- Baking visual information from the source scan.
- Saving texture maps.
- Preparing assets for further editing or export.

This can be useful for rocks, statues, props, terrain pieces, objects, ruins, artifacts, and real-world captured surfaces.

---

## Large Scan Cleanup

Large scans can slow down Blender and make editing difficult.

ScanReady 1.0 helps create a more manageable asset that is easier to:

- Orbit in the viewport.
- Inspect.
- Bake.
- Export.
- Use in scenes.
- Share with collaborators.

The optimized result can be used as a lighter production asset while keeping the original high-poly scan as the source.

---

## Typical Workflow

A typical ScanReady 1.0 use case looks like this:

1. Import or open a high-poly scan in Blender.
2. Select the scan object.
3. Use **ONE CLICK BAKE** for a fast automatic result.
4. Inspect the optimized mesh and baked textures.
5. Use manual steps if the asset needs more control.
6. Export the final asset for VR, games, realtime visualization, or presentation.

---

## Main Benefit

ScanReady 1.0 helps bridge the gap between scan quality and realtime usability.

It turns heavy captured models into assets that are easier to manage, easier to bake, and easier to use in production.

The result is a lighter asset that keeps the look of the scan without carrying all the original geometry weight.
